PART 1
보편적 아키텍처

모든 엔지니어링 도메인을 통관하는 불변의 동작 흐름

모든 서비스의 동작 생명주기 Slide 02

게임, 웹, 모바일 서비스는 본질적으로 모두 동일한 4단계 흐름을 공유합니다.

Input (입력)
브라우저 마우스 클릭 / 소켓 패킷
Process (처리)
비즈니스 가공 / 연산 및 유효성 검증
Store (저장)
데이터베이스 영속적 파일 기록
Output (응답)
성공 상태 전송 / 화면 UI 랜더링

도구의 고유 명법에 구애받지 않고 '데이터 흐름'의 안정을 설계하는 것이 엔지니어의 핵심 본분입니다.

서비스 도메인별 실무 동작 흐름 Slide 03
구분 Input (요청) Process (처리) Store (저장)
웹 (Web Development) 강의평가 등록 버튼 누름 욕설 필터링 및 작성 수강자 대조 평가 정보 및 점수 DB 기록
게임 (Game Development) 몬스터 공격 스킬 입력 데미지 공식 연산 및 쿨타임 체크 캐릭터 스탯 및 아이템 정보 갱신
앱 (App Development) 배달 실시간 현황 갱신 위치 좌표 동기화 및 잔여 시간 연산 실시간 위치 기록 갱신
DevOps 파이프라인 소스 코드 push 수행 코드 컴파일, 테스트 빌드 자동화 빌드 이력 및 패키징 아티팩트 보관
계층 격리: 3-Tier Architecture Slide 04

유지보수 확장성을 보장하기 위해 각 모듈의 책임을 분리하여 구축하는 기전입니다.

식당 비유를 통한 계층 이해

  • Presentation (테이블): 사용자와 접하는 공간. React 프론트엔드.
  • Application (주방): 음식을 요리(연산)하는 공간. NestJS WAS 서버.
  • Data Layer (창고): 재료를 신선하게 보관하는 공간. MySQL DB.
Presentation Tier (React UI)
Application Tier (NestJS Web Application Server)
Data Tier (MySQL Database)
서버 아키텍처: Web Server vs WAS Slide 05

웹 서버 (Web Server)

  • 대표 기술: Nginx, Apache
  • 동작 특징: 정적 컨텐츠(HTML, JS, CSS, 이미지) 초고속 분배 처리.
  • 실무 역할: 최전방에서 요청 수신, 정적 캐싱 서빙 및 프록시 방패막이.

웹 애플리케이션 서버 (WAS)

  • 대표 기술: NestJS, Java Spring, Tomcat
  • 동작 특징: 동적 연산(로그인 처리, 인가 가드, DB 비즈니스 로직).
  • 실무 역할: 기밀 처리가 일어나는 프라이빗 존에 은닉 배치되어 연산 집중.

분산 형상 관리

분산 형상 관리 시스템(DVCS)의 정체 Slide 08

중앙집중식 VCS (SVN 등)

중앙 서버 한곳에 전역 소스코드가 있어 인터넷 연결이 유실되거나 중앙 서버가 폭파되면 이력 파악이 완전히 불가능해집니다.

분산식 VCS (Git)

원격지 인증: HTTPS vs SSH Slide 09
비교 항목 인증 원리 및 기전 실무 관점 장단점
HTTPS ID, 개인 패스워드 혹은 Personal Access Token 기반 전송 단발성 접근이 매우 편하나 인증 유효기간 만료 시 재생성 번거로움
SSH 로컬 개인키와 GitHub 공개키 대조 비대칭키 암호화 기전 최초 1회 등록 시 반영구적인 무인증 SSH 통신 터널 개방 (추천)

자동화 배포 환경(CI/CD) 수립을 위해서는 비밀번호 입력 과정을 원천 스킵해야 하므로 SSH 키 등록이 실무 표준입니다.

SSH 키 생성 및 비대칭 키 인증 Slide 10
SSH Key 발급 기전

PART 3
Frontend & React SPA

가상 DOM 렌더링 최적화 알고리즘과 번들링 라이프사이클

브라우저의 화면 가공 지도: DOM Slide 13

DOM (Document Object Model)

  • 브라우저가 텍스트 HTML을 컴파일해서 만드는 메모리 상의 객체 지도.
  • 화면 디자인에 변화가 생기면 DOM 노드가 갱신되며 리렌더링 유도.
  • DOM을 직접 자주 건드리는 조작은 레이아웃 재연산(Reflow/Repaint) 부하를 유발하는 고비용 작업.
단 한 장의 웹페이지의 마법: SPA Slide 14

전통 웹페이지 (MPA)

메뉴 클릭마다 서버에 페이지 전역 HTML을 매번 새로 요구하여 깜빡거림 발생 및 연산 낭비 유발.

현대적 싱글 페이지 웹 (SPA)

단 한 장의 index.html 위에서 자바스크립트가 브라우저 메모리를 감쪽같이 스와핑하여 컴포넌트를 무깜빡임 고속 교체.

React 성능의 핵심: 가상 DOM Slide 15

리액트는 실제 브라우저 무거운 DOM을 매번 수정하는 낭비를 막기 위해 메모리 사본인 가상 DOM(Virtual DOM)을 활용합니다.

Diffing & Reconciliation

  • 1. 데이터 변동 감지 시, 가상 메모리 공간에 가상 DOM 사본 우선 렌더링.
  • 2. 이전 메모리 사본과 새 사본의 구조적 비교 대조(Diffing).
  • 3. 정말 변동이 일어난 딱 한 영역의 노드만 선별 추출.
  • 4. 실제 화면 브라우저 DOM에는 오직 그 바뀐 노드만 패치(Reconciliation).
React 반응성 원천: State & Hooks Slide 16

useState (상태 추적)

화면 드로잉과 밀착 동기화된 가반응 데이터 주머니입니다. 일반 변수와 달리 값이 수정되는 순간, 리액트가 리렌더링 연산을 동기 가동하게 설계되어 있습니다.

useEffect (부수 효과)

화면 랜더링이 완벽히 완수된 시점 이후에 수반되어야 하는 비동기 데이터 API 통신, 타이머 실행 등 '사이드 이펙트' 제어를 통제하는 안전 관리자입니다.

비동기 데이터 흐름제어: async / await Slide 17

프론트엔드 디버깅 & 소스맵

F12 통신 검문소 설계와 소스코드 유출 차단 보안 대응

크롬 개발자 도구(F12) 네트워크 검문 프로토콜 Slide 20

1. Request (보낸 패킷)

Payload 영역의 JSON 데이터 속성에 불필요한 누락이나 오타 필드가 엮이진 않았는지 최종 송신 정보 대조.

2. Status Code (상태 검문)

404는 주소 오류, 500은 백엔드 서비스 가동 중단, 403은 인가 차단 상태 등의 흐름을 코드 번호로 사냥 검식.

3. Response (받은 패킷)

서버 엔진이 정상 가공해 리턴한 JSON 포맷 데이터를 확인하고 프론트 변형 바인딩 가능 여부 최종 대조.

실무 단골 차단 에러 1위: CORS 정책 위반 Slide 21

CORS 차단 메커니즘

브라우저는 사용자 악성 스크립트 도용 차단을 위해, 3000포트(리액트)에 로드된 자바스크립트 코드가 4000포트(NestJS)로 데이터 전송을 감행하는 포트 불일치 시 통신을 임의 불허합니다.

해결 실무 코드

백엔드 서버 셋업 소스 코드 최전방 진입점에 app.enableCors() 처리를 하여, 브라우저에 공인 매칭 검증 서명을 주입해 빗장을 열어줍니다.

압축 번들링의 숨겨진 지도: Source Map Slide 22

Minify / Obfuscation

소스맵의 정체

난독화되어 알 수 없는 외계어 코드의 에러 위치를 원래 코딩했던 오리지널 App.tsx 파일의 80번째 라인이라고 복원 대치해주는 기밀 이정표 맵 파일입니다.

실무 소스맵 유출 방지 및 보안 대응 Slide 23

보안 무단 탈취 위험성

백엔드 NestJS & 3계층

비즈니스 로직 격리와 제어의 역전(IoC) 의존성 바인딩

백엔드의 신뢰 영역: 보안과 기밀 가치 Slide 26

비신뢰 영역 (Untrusted Area)

사용자 브라우저 기기(React)는 데이터 조작이 손쉬워 아무런 보안이 담보되지 않는 최전선 정찰 구역입니다.

백엔드 기밀 성벽 (Security Boundary)

기밀 DB 패스워드와 환경 암호 키를 철저히 물리적으로 숨겨 두고 외부 망과의 직접 연결을 차단하는 인프라 수호의 핵심 성벽입니다.

백엔드 기술 스택 지평 조망 Slide 27
기술 명세 사용 언어 핵심 장점 및 실무 포지션
Java Spring Java 엄격한 OOP 원칙 준수, 대규모 엔터프라이즈의 표준 견고성
NestJS TypeScript JS 생태계 고성능 비동기 처리 기전과 Spring의 정교한 설계 철학 병합
Python Django Python 생산성이 극도로 뛰어남, 풍부한 AI 라이브러리 확장성
Go Gin Go 극도로 빠른 컴파일 속도, 마이크로서비스 경량 분산 처리 최적
의존성 주입(DI)과 제어의 역전(IoC) Slide 28

의존성 주입 (DI)

내가 스스로 코드로 `new` 연산자를 써서 단단히 부품을 결착해 조립해버리지 않고, 뒷단에서 부품을 받아다가 꽂아 사용하도록 조립 공간을 유연화하는 방식입니다.

제어의 역전 (IoC)

부품 생성 주도권(제어권)이 개발자 손을 떠나 WAS 컴파일 컨테이너로 넘어가는 설계 전환이며, 덕분에 모듈 단위 조각 테스트가 용이해집니다.

데이터베이스 & 실전 복구

RDBMS 무결성 모델링 설계와 현업 장애 극복 트러블슈팅

데이터 보관소 대결: RDBMS vs NoSQL Slide 33

관계형 DB (RDBMS)

  • MySQL, PostgreSQL.
  • 동작 성격: 엄격한 엑셀 표 형태 및 외래키 결합 규격.
  • 존재 가치: 금융 결제, 회원 데이터 등 '단 한 자의 정합성 오류'도 차단해야 하는 신뢰 데이터 담당.

비관계형 DB (NoSQL)

  • MongoDB, Redis.
  • 동작 특징: 스키마 없이 자유로운 형식의 도큐먼트 서류 뭉치.
  • 존재 가치: 실시간 고속 채팅, 대규모 트래픽 로그 및 세션 분산 저장 기전 담당.
데이터 무결성 지킴이: ACID 원칙 Slide 34

관계형 데이터베이스(MySQL)가 데이터의 영구적 안정 보장을 위해 수호하는 4대 기본 법칙입니다.

ACID 상세 물리 규칙

  • Atomicity (원자성): 다 저장되거나 다 터지거나 (중간에 내 돈만 깎인 채 통신 중단 불가).
  • Consistency (일관성): 사전에 부합된 조건 제약(수량은 무조건 0이상 등) 절대 충족 보장.
  • Isolation (고립성): 여러 사람이 동시에 조회/수정 요청 시 순차 줄세우기 처리 꼬임 소멸.
  • Durability (지속성): 기록 완료 후 정전이 되어도 물리 하드웨어 디스크 파일 영구 안전화.
객체로 쿼리 변환: ORM vs SQL Mapper Slide 35
비교 도구 동작 제어 기전 실무 장단점
SQL Mapper (MyBatis) 백엔드 코드 상에 로우 쿼리를 직접 타이핑 결합 후 DB 송신 수동 정밀 튜닝 최적화 유리하나 코드 장황화 발생
ORM (TypeORM / JPA) 데이터베이스 테이블을 코드 상의 클래스 객체와 1대1 본딩 메소드 호출 시 백그라운드 쿼리 동적 오토 빌드 (엄청난 생산성)
테이블 정규화와 1:N 외래키 관계 설계 Slide 36

테이블 간의 관계(Relation)를 통제하여 중복 데이터를 배제하고 무결성을 획득하는 수식 설계도입니다.

강의평가 1:N 관계 명세

  • Lectures (1) : Evaluations (N)
  • 하나의 명강의 카드 밑에는 수많은 수강생의 평가 조각들이 수록 결합됩니다.
  • On Delete Cascade: 강의가 삭제되면 그 아래 작성된 평가 글도 시스템 일치성을 위해 자동 전수 영구 증발 소멸 연계 처리.
